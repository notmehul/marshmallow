# Warm Interface Reference

## Rule

Use warm, restrained, conversational interfaces as the quality anchor when
several competent design directions are possible.

## Evidence

- `like-warm-interface` - the reference emphasizes clear hierarchy, restrained
  controls, no dashboard clutter, and an emotional contract that feels like a
  focused helper rather than an operations console.

## Use In Skills

- `frontend-design` - use warmth and restraint as defaults for first-pass visual
  direction.

## Limits

Use a different anchor when the user explicitly asks for dense operations,
editorial spectacle, or a colder enterprise surface.

## Connections

- [[helper-not-dashboard]] - this reference supplies the visual anchor for the
  helper-like emotional contract.
