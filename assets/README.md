# Brand assets

Marshmallow follows the "Go gopher" strategy: a cute, trustable mascot —
**Marshy** — in front of a deliberately robust system. The art is the persona;
the trust model is the substance.

## Drop-in slots

The README references these files. Commit the PNGs here and they render
automatically — until then the README falls back to text and badges.

| File | Used by | Suggested art |
| --- | --- | --- |
| `marshy-hero.png` | README hero | Marshy + "Make your AI agents less generic" wordmark |
| `marshy-how-it-works.png` | (optional) How it works | sources → context → agent → better output strip |
| `marshy-trust.png` | (optional) No silent memory tricks | local-first / no-log-hoarding / rollback trio |

To enable the hero image, replace the comment at the top of `README.md` with:

```html
<p align="center"><img src="assets/marshy-hero.png" alt="Marshy, the Marshmallow mascot" width="640"></p>
```

## Naming

- **Marshmallow** — the project / plugin / CLI.
- **Marshy** — the mascot. (Earlier sketches used "Marsh"; the project name is
  always the full "Marshmallow".)

## Style

Hand-drawn, warm, off-white character with glasses and a green check. Keep
backgrounds clean. Orange is the accent; the green check signals "trusted /
done." Avoid glossy gradients and dashboard chrome — the visual language should
match the product's restraint.
